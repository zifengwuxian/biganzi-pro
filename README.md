# 机关笔杆子 Pro

基于 DeepSeek-V3 商业版模型的智能公文润色工具，帮助体制内工作者快速生成高质量的公文材料。

## 功能特点

- ✅ **一键润色周报** - 将口语化的周报快速转换为正式公文格式
- ✅ **口语秒变公文** - 将日常口语表达转换为规范的公文语言
- ✅ **自动扩写汇报材料** - 基于简短素材自动扩写为完整的汇报材料
- ✅ **专业笔杆子风格** - 模拟20年工龄体制内资深"笔杆子"的写作风格
- ✅ **结构化输出** - 自动按照"一、二、三"的结构组织内容
- ✅ **词汇升级** - 自动使用更正式、更专业的公文词汇
- ✅ **政治站位高** - 确保内容符合公文的政治要求

## 技术栈

- **前端框架**: Streamlit
- **AI模型**: DeepSeek-V3 (商业版)
- **API客户端**: OpenAI SDK
- **编程语言**: Python 3.x

## 安装部署

### 1. 环境准备

```bash
# 克隆项目
git clone <repository-url>
cd Wordsmith Pro

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 配置密钥

在项目根目录创建 `secrets.toml` 文件，并填入以下配置：

```toml
GITHUB_TOKEN = "your_github_token"
GIST_ID = "your_gist_id"
ZHIPU_KEY = "your_zhipu_key"
DEEPSEEK_KEY = "your_deepseek_api_key"
```

**重要提示**：
- `DEEPSEEK_KEY` 是必需的，用于调用 DeepSeek API
- 其他密钥用于授权验证系统（可选）
- 请妥善保管密钥，不要提交到版本控制系统

### 4. 运行应用

```bash
streamlit run app.py
```

应用将在浏览器中自动打开，默认地址为 `http://localhost:8501`

## 使用说明

### 基本使用

1. **解锁应用**
   - 在左侧边栏输入访问密码
   - 扫描二维码或添加微信联系获取密码

2. **输入素材**
   - 在文本框中输入需要润色的原始内容
   - 可以是口语化的表达、简短的汇报要点等

3. **生成公文**
   - 点击"立即润色"按钮
   - 等待 AI 处理（通常几秒钟）
   - 查看生成的公文内容

### 使用示例

**输入示例**：
```
去社区看了看，发现卫生很差，垃圾到处都是，居民意见很大。
```

**输出示例**：

```
经实地走访调研发现，社区卫生环境存在突出问题。

二、问题分析
（一）垃圾清运不及时，存在堆积现象
（二）环境卫生管理机制不完善
（三）居民反映强烈，满意度偏低

三、工作建议
（一）加强日常保洁频次
（二）完善垃圾清运体系
（三）建立长效管理机制
```

## 核心代码说明

### app.py - 主应用

- **核心配置**: 从 `secrets.toml` 读取 API 密钥
- **商业闭环**: 实现密码验证和收费逻辑
- **AI润色**: 调用 DeepSeek API 进行文本润色

### auth.py - 授权模块

- **卡密验证**: 基于 GitHub Gist 的云端授权系统
- **多应用支持**: 支持多个应用共用一套授权系统
- **状态管理**: 支持卡密激活、过期检查等功能

```
你是一位拥有 20 年工龄的体制内资深"笔杆子"。
```

## 注意事项

1. **API密钥安全**
   - 不要将 `secrets.toml` 提交到公开仓库
   - 定期更换 API 密钥
   - 使用环境变量或密钥管理服务

2. **使用限制**
   - 需要有效的访问密码才能使用完整功能
   - API 调用可能产生费用，请注意使用量

3. **内容质量**
   - 输入内容越具体，生成质量越高
   - 建议提供足够的背景信息和要点
   - 生成后需要人工审核和调整

4. **部署建议**
   - 本地开发：直接运行 `streamlit run app.py`
   - 云端部署：推荐使用 Streamlit Cloud
   - 生产环境：建议配置域名和 SSL 证书

## 常见问题

### Q: KeyError: This app has encountered an error
A: 检查 `secrets.toml` 中的密钥名称是否与代码中的引用一致。确保使用 `DEEPSEEK_KEY` 而不是 `DEEPSEEK_API_KEY`。

### Q: API 调用失败
A: 检查以下几点：
- API 密钥是否有效
- 网络连接是否正常
- API 服务是否可用
- 是否超出调用限制

### Q: 生成质量不理想
A: 尝试以下方法：
- 提供更详细的输入内容
- 调整系统提示词
- 多次生成选择最佳结果
- 人工后期润色
